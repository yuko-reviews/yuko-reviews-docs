The Yuko Integration plugin exposes WordPress hooks so you can customize its behaviour without forking the plugin. All hooks live in the global namespace and are prefixed with `yuko_`.

<Info>
  Add custom code to your child theme's `functions.php` or a site-specific MU plugin. Never edit the Yuko plugin files directly — changes are overwritten on update.
</Info>

---

## Action hooks

Fired at key points in the plugin lifecycle.

| Hook | Fires | Use case |
| --- | --- | --- |
| `yuko_before_init` | Before `Route::init()` registers components | Replace or wrap a component before the default is registered |
| `yuko_after_init` | After all components are registered | Enqueue supplementary scripts, register additional UI |
| `yuko_before_schema_output` | Just before JSON-LD product schema is printed | Modify or extend the schema markup |
| `yuko_clear_cache` | When the admin triggers a cache reset or after certain REST updates | Clear your own caches that depend on Yuko data |
| `yuko_reactivate_disabled_webhooks` | Scheduled recovery action for disabled webhooks | Hook in to log recoveries or notify monitoring |

**Example — enqueue a companion script after init:**

```php
add_action( 'yuko_after_init', function () {
    wp_enqueue_script(
        'my-yuko-customizations',
        get_stylesheet_directory_uri() . '/js/yuko-custom.js',
        [ 'yuko-review-main' ],
        '1.0.0',
        true
    );
} );
```

## Filter hooks

Modify plugin values before they are used.

### Webhook delivery

| Filter | Default | Purpose |
| --- | --- | --- |
| `yuko_webhook_topics` | Registered topic array | Add or remove webhook events the plugin fires |
| `yuko_blocked_order_statuses_for_webhooks` | `['draft', 'auto-draft']` (configured at runtime) | Prevent certain order statuses from firing webhooks |
| `yuko_order_cancelled_status` | `'cancelled'` | Map a custom WooCommerce status to Yuko's "cancelled" event |
| `yuko_order_fulfilled_status` | `'completed'` | Map a custom status to the "fulfilled" event |
| `yuko_order_paid_status` | `'processing'` | Map a custom status to the "paid" event |
| `yuko_max_webhook_delivery_failures` | Runtime default | Override the retry cap before a webhook is auto-disabled |
| `yuko_product_update_time_period` | Runtime default | Change the window used to debounce rapid product update webhooks |

**Example — add a custom webhook topic:**

```php
add_filter( 'yuko_webhook_topics', function ( $topics ) {
    $topics['order.custom_shipped'] = [
        'label' => 'Order marked as shipped',
        'hook'  => 'woocommerce_order_status_shipped',
    ];
    return $topics;
} );
```

### Storefront rendering

| Filter | Purpose |
| --- | --- |
| `yuko_disable_woocommerce_ratings` | Return `true` to hide WooCommerce's native star ratings (Yuko replaces them by default) |
| `yuko_enable_category_page_rating_widget` | Return `true` / `false` to toggle star display on category and archive pages |
| `yuko_structured_data_product` | Modify the JSON-LD product schema array before output |
| `yuko_product_sync_status` | Array of post statuses that sync to Yuko (default: published only) |

**Example — show Yuko ratings on category pages:**

```php
add_filter( 'yuko_enable_category_page_rating_widget', '__return_true' );
```

### REST and admin behaviour

| Filter | Purpose |
| --- | --- |
| `yuko_bulk_meta_max_batch` | Cap the batch size when Yuko issues bulk product-meta updates |
| `yuko_wordpress_disconnect_url` | Override the URL the plugin hits during the disconnect flow |

## Plugin constants

Defined in the main plugin file. You can read these but should not override them.

| Constant | Purpose |
| --- | --- |
| `YUKO_PLUGIN_NAME` | Plugin display name |
| `YUKO_VERSION` | Current plugin version |
| `YUKO_PLUGIN_PATH` | Absolute filesystem path to the plugin directory |
| `YUKO_PLUGIN_URL` | Public URL of the plugin directory |
| `YUKO_PLUGIN_FILE` | Absolute path to the main plugin file |
| `YUKO_PLUGIN_SLUG` | Slug used in option keys |
| `YUKO_PREFIX` | Prefix used for option keys, hooks, and transients |
| `YUKO_WC_REQUIRED_VERSION` | Minimum WooCommerce version the plugin requires |
| `YUKO_REQUIRED_PHP_VERSION` | Minimum PHP version |
| `YUKO_REQUIRED_WP_VERSION` | Minimum WordPress version |
