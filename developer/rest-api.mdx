The Yuko Integration plugin registers WordPress REST endpoints under the `yuko/v2` namespace. These are used internally by the Yuko backend to sync data and by merchant scripts for advanced automation.

<Info>
  Prefer **[Webhooks](/integrations/webhooks)** or **[Zapier](/integrations/zapier)** for no-code automation. The REST endpoints below are intended for custom integrations that need direct access to the plugin.
</Info>

---

## Authentication

All requests require a WooCommerce REST API key with appropriate permissions:

```bash
curl -u consumer_key:consumer_secret \
  https://yourstore.com/wp-json/yuko/v2/getStoreData
```

Generate keys in **WordPress Admin → WooCommerce → Settings → Advanced → REST API**.

---

## Endpoints

All endpoints live under `https://yourstore.com/wp-json/yuko/v2/`.

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/handshake` | Initial authentication handshake between the plugin and the Yuko backend |
| `POST` | `/reviews/settings` | Update review-related settings on the WP side |
| `GET` | `/getStoreData` | Return store information (name, URL, WC version, plugin version) |
| `POST` | `/disconnect` | Disconnect the store from Yuko |
| `POST` | `/updateProductMeta` | Update Yuko metadata on a single product |
| `POST` | `/reviews/updateBulkProductMeta` | Bulk update Yuko metadata across products (batch size capped by the `yuko_bulk_meta_max_batch` filter) |
| `POST` | `/update/content` | Update storefront content cache |
| `POST` | `/coupons` | Create a coupon in WooCommerce for a review reward |
| `POST` | `/customer/purchased/history` | Return purchase history for a customer (used to verify buyer status) |

Each endpoint validates the request against the stored `organization_uid` and a HMAC-SHA256 signature sent in the request headers.

---

## Common integration patterns

### Verify a customer's purchase history

Use `/customer/purchased/history` to check whether an email has purchased a product before awarding a verified badge, or to gate a review form.

### Create a reward coupon programmatically

The `/coupons` endpoint lets your own automation issue review-reward coupons outside of Yuko's built-in workflows.

### Bulk recalculate product metadata

After importing reviews or adjusting display rules, call `/reviews/updateBulkProductMeta` to refresh the star ratings and review counts stored on each WooCommerce product.

## Extending the plugin's REST surface

If you need an endpoint that isn't listed above, register your own inside the `init` action using WordPress's `register_rest_route()` — the Yuko plugin also exposes action hooks you can piggyback on. See [Plugin Hooks & Filters](/developer/plugin-hooks).

---

## Reading review data from Yuko

Yuko's SaaS backend exposes a small **public API** that is currently focused on loyalty, rewards, and customer programs — features available on the Shopify platform. WooCommerce merchants who need to read review data from an external service should use:

1. **[Webhooks](/integrations/webhooks)** to push review events to your own endpoint as they happen
2. **[Klaviyo integration](/integrations/klaviyo)** to route review events into a marketing platform
3. **[Zapier](/integrations/zapier)** for no-code automation against hundreds of apps

A public review-reading API for WooCommerce is on our roadmap. [Talk to us](https://yukoreviews.com/support) if you have a specific use case — we prioritize based on real demand.
